# Contributing

We welcome contributions! Please follow these guidelines:

## Pull Request Process

1.  Fork the repository.
2.  Create a new branch for your feature or bug fix.
3.  Ensure your code follows the existing style and standards.
4.  Run all verification steps before submitting your PR.
5.  Provide a clear description of your changes and any relevant verification evidence.

## Issue Reports

Please use the [GitHub issues](https://github.com/metyatech/skill-post-deploy/issues) to report bugs or suggest features.

Include as much detail as possible in your reports, including:
- Steps to reproduce
- Expected behavior
- Actual behavior
- Screenshots or logs if applicable
- Your environment details
